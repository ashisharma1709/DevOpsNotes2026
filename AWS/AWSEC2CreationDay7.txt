************************  Create First EC2 instance without key pair using IAM user **************************

* What is EC2 ?

EC2 = cloud ka computer
Matlab AWS pe ek virtual machine milti hai
Usme tum Ubuntu install karke use kar sakte ho jaise apna PC


🚀 Step-by-step: 

1. AWS Console open karo, and login as IAM user safte option
👉 https://console.aws.amazon.com/
Search: EC2 → open karo

2. Launch Instance pe click karo

3. Name do

Kuch bhi simple naam: MyFirstServer

4. OS select karo: Ubuntu Server 22.04 LTS

5. Instance type: t2.micro (Free tier, paisa nahi lagega)

6. Key Pair (IMPORTANT): Select: Proceed without key pair

7. Network settings
Edit pe click karo aur:
✅ Public IP → Enable
✅ SSH → allow (0.0.0.0/0)
✅ HTTP → allow (future ke liye)


8. Storage: Default rehne do (8GB)

9. Launch Instance
Click karo → Launch
 1–2 min wait karo

🔑 Connect kaise kare (without key pair)
EC2 dashboard → Instances
Apna instance select karo
Click Connect

Select:
👉 EC2 Instance Connect
Click Connect
Terminal open ho jayega

Result: abhi ek live server bana liya, Internet pe available hai
Isme:
website host kar sakte ho
backend bana sakte ho
APIs run kar sakte ho


*************************  Access Your Machine from browser once its up through public IP **************************

Jab humne public IP dala → “refused to connect” kyun aaya?

👉 Simple reason:Server chal raha tha, lekin koi application / service run nahi ho rahi thi jo request handle kare.

Matlab:
EC2 (Ubuntu machine) = ON ✅
Lekin uske andar koi web server/app nahi tha ❌

Isliye browser bola:“bhai yahan koi sun hi nahi raha” → connection refused

💡Phir nginx start kiya :
sudo apt install nginx
sudo systemctl start nginx

tab kya hua:👉 Nginx ne port 80 (HTTP) pe listen karna start kiya

Ab jab browser me IP dala:
Request gayi → EC2 pe
Nginx bola → “haan bhai, main hoon 😎”
Aur ek default page dikha diya

🔸 EC2 kya hai: Sirf ek empty computer (server)

🔸 Nginx kya hai: Ek web server software


Real-life example
EC2 = ghar
Nginx = darwaza + guard

Case 1: (Nginx nahi tha)
Ghar hai, par darwaza hi nahi
👉 koi andar kaise aayega? → ❌ refused

Case 2: (Nginx start kiya)
Ab darwaza + guard hai
👉 koi aaya → entry mil gayi → ✅ page dikha


Nginx actually karta kya hai?

Nginx ek powerful web server hai jo:
HTTP request handle karta hai
Web pages serve karta hai
Reverse proxy ka kaam karta hai
Load balancing bhi karta hai (advanced)


Browser (tum) 
   ↓
Public IP
   ↓
EC2 server
   ↓
Nginx (port 80 pe sun raha hai)
   ↓
Response (web page)


Agar Nginx install hai but start nahi:
sudo systemctl status nginx

sudo systemctl start nginx

👉 EC2 = machine
👉 Nginx = web server jo internet se baat karta hai
