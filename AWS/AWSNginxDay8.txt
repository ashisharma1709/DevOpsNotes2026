********************** What is NGINX?  *******************************

Nginx kya hai?
Nginx ek ek web server software hai hai jo internet se aane wali requests ko handle karta hai aur browser ko response deta hai
“Engine-X” = “Engine” → powerful system, “X” → scalable / unknown large scale

👉 Nginx = web server (jo website ko browser tak pahunchata hai)
 Nginx = software (web server)
 Running Nginx = service


Flow:

Jab tum browser me likhte ho: http://your-ip

Toh:Browser → request bhejta hai → Nginx → file/app → response → browser


Nginx ka kaam kya hai?

1️. Website dikhana: HTML, CSS, JS serve karta hai

2️. Requests handle karna: Kaun kya access karega decide karta hai

3️. Backend ke paas request bhejna: Java / Node app ko forward karta hai

Real life example
Internet = road
Tumhara EC2 = ghar
Nginx = gatekeeper / guard 🚪

👉 Koi aaya:
Guard poochta hai → kya chahiye?
Fir andar bhejta hai ya response deta hai
❌ Nginx nahi ho to

👉 Request aayegi → koi handle nahi karega
👉 Result = connection refused

✅ Nginx ho to

👉 Request aayegi → Nginx pakdega
👉 Page ya response bhej dega

Nginx = software jo port 80/443 pe baith ke requests sunta hai aur response deta hai



* DevOps usecase:

Example:

********************************  Reverse Proxy?  ********************************************

Jab user directly backend se baat nahi karta,  Beech me Nginx hota hai
Nginx request ko receive karta hai aur backend app ko forward karta hai

Without Reverse Proxy (direct access)
User → Backend (Spring Boot :8080)

❌ Problems:
Port expose hota hai (8080), Security kam, Load handle nahi hota

With Reverse Proxy (Nginx)
User → Nginx (port 80) → Backend (port 8080)

User ko sirf Nginx dikhega, Backend hidden rahega

Practical setup:

Step 1: Backend app run karo. Example (Spring Boot): 
java -jar app.jar
Ye chalega:http://localhost:8080

Step 2: Nginx install (agar nahi hai): sudo apt install nginx -y

Step 3: Nginx config edit karo: sudo nano /etc/nginx/sites-available/default

server {
    listen 80;

    location / {
        proxy_pass http://localhost:8080;
    }
}

Step 4: Nginx restart: sudo systemctl restart nginx

Tum browser me sirf: http://<your-public-ip> hit karo

Flow kya hoga internally

User (browser)
   ↓
Nginx (port 80)
   ↓
Forward to localhost:8080
   ↓
Spring Boot app
   ↓
Response back via Nginx
   ↓
Browser pe output


Result: Tu port 8080 hide kar diya, Ab sab traffic Nginx se aa raha hai
