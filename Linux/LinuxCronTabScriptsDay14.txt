*************** Crontab with Scripts ****************

💻 1. Script ke saath kaise use kare :
	
	🔥 Step 1: Script banao
	nano backup.sh
	#!/bin/bash
	echo "Backup started"
	date >> /home/ubuntu/backup.log
	
	🔥 Step 2: Permission do
	chmod +x backup.sh
	
	🔥 Step 3: Cron mein add karo
	crontab -e

	Add:
	0 2 * * * /home/ubuntu/backup.sh >> /home/ubuntu/output.log 2>&1
	
	🎯 Result:

	👉 Roz 2 baje script chalegi
	👉 Output + error log file mein save hoga
	
	

🚀 2. Script kahaan use hoti hai?

	🔥 Real DevOps Use Cases:
	✅ Backup automation
	0 1 * * * backup.sh
	✅ Log cleanup
	0 0 * * * cleanup.sh
	✅ Health check
	*/5 * * * * health.sh
	✅ DB dump
	0 3 * * * db_backup.sh	
	
Script likha → permission diya → crontab -e → schedule likha → cron run karega automatically	

⚠️ 3. Important Rules:

	✔ Always full path use karo
	/home/ubuntu/script.sh
	
	✔ Always logging lagao
	>> log.log 2>&1
	
	✔ Script executable hona chahiye
	chmod +x script.sh
	
	✔ Cron service running hona chahiye
	systemctl status cron
	
🧪 4. Run kaise check kare?
	🔹 Logs check:
	grep CRON /var/log/syslog
	🔹 Manual run:
	sh /home/ubuntu/script.sh	
	
	
💥 4. Ek Full Real Example
🎯 Requirement:

	👉 Har 10 min health check
	👉 Output save karo

	Script:
	#!/bin/bash
	curl -I google.com

	Cronjob:
	*/10 * * * * /home/ubuntu/health.sh >> /home/ubuntu/health.log 2>&1	
	
	

🚀 Short

👉 "Cron ek Linux scheduler hai jo background mein run hota hai.
Crontab ek configuration file hai jisme hum schedule define karte hain,
aur cronjob actual command hota hai jo scheduled time pe execute hota hai.
Hum scripts ko automate karne ke liye cron use karte hain jaise backup, monitoring, cleanup
