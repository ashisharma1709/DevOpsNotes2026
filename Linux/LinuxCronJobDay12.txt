**********  CRON **************
Cron is a job scheduling utility present in Unix like systems.
It reads the crontab for running predefined scripts. By using a specific syntax, we can configure a cron job to schedule scripts or other commands to run automatically.


👉 Crons vs Crontab vs Cronjob

👉 Cron is a daemon that runs at the times of system boot.

👉 Crontab is a file that contains the schedule of cron entries to be run and at specifc times. File location varies by OS.

👉 Cronjob or cron schedule is a specific set of executionof instructions specfiying day, time and command by execute.

* * * * * /path to file/script.sh
* * * * * /path to file/script.sh >> /tmp/test.txt

Crontab commands :

crontab -e : edit crotab file, or create one if doesn't already exists.
crontab -l : list of cronjobs, display crontab file contents.
crontab -r : remove crotab file
crontab -v : Display the last time we edited our crontab file.(this option available only in few systems)
systemctl status cron -> Cron service status check



****************  In More Simple term *****************************

👉 Cron = background service (daemon)
		  Linux ka scheduler hai
		  Automatically tasks run karwata hai specific time pe

		📌 Example:

			Roz 2 baje backup lena
			Har 5 min logs check karna
			Sunday ko server restart


👉 Crontab = configuration file

			Isme hum schedule likhte hain
			Kaunsa command kab chalega

			Command:

			crontab -e

			👉 Ye open karega ek file jahan hum schedule likhoge


👉 Cronjob = actual task

			📌 Example:

			* * * * * echo "Hello DevOps"

			👉 Ye ek cronjob hai


🧩 Structure :
* * * * * command
| | | | |
| | | | └── Day of week (0-7) (Sunday = 0/7)
| | | └──── Month (1-12)
| | └────── Day of month (1-31)
| └──────── Hour (0-23)
└────────── Minute (0-59)


*******************  Examples of cronjob  *****************************

✅ Har minute run kare   -> * * * * * command


✅ Har 5 min  -> */5 * * * * command


✅ Roz 2:30 AM  -> 30 02 * * * command

✅ Har Sunday   -> 0 0 * * 0  command

✅ Har month 1st date  -> 0 0 1 * * command


**************************  Real Use cases ***************************

💻 Real DevOps Use Cases

🔥 1. Log cleanup
0 0 * * * rm -rf /var/log/*.log

0 1 * * * /home/ubuntu/backup.sh

*/5 * * * * curl -I google.com

********   Cron Parameters *********************

🧠 Step 1: Cron is like time clock

	Cron ka format hota hai:

	Minute  Hour  Date  Month  Day

	👉 Total 5 boxes hote hain

	📦 Box 
	[ 1 ] [ 2 ] [ 3 ] [ 4 ] [ 5 ]
	  M     H     D     M     W
	Box	Meaning
	1	Minute
	2	Hour
	3	Date (month ka din)
	4	Month
	5	Week day


	🔥 Rule :

	👉 * = "har value"
	👉 */5 = "har 5 step"
	👉 0 = exact value



❌ Example 1:
	*/5 * * * * command

	👉 Isko break karte hain:

	Box	   Value	Meaning
	Minute	*/5		har 5 min
	Hour	*		har hour
	Date	*		har din
	Month	*		har month
	Day		*		har weekday
	
	✅ Final Meaning:

	👉 Har 5 minute mein command chalega

	🕐 Example:

	12:00
	12:05
	12:10
	12:15


❌ Example 2:
	0 0 * * 0 command   

	Break karte hain:

	Box		Value	Meaning
	Minute	0		minute 0   exact start of hour
	Hour	0		hour 0   12 AM (midnight)
	Date	*		har din
	Month	*		har month
	Day		0		Sunday
	
	✅ Final Meaning:

	👉 Har Sunday raat 12 baje (midnight)

	🕐 Example:

	Sunday 00:00
	
💥 Shortcut Trick : Minute → Hour → Date → Month → Day	


*/10 9-17 * * 1-5    -> 9 to 5 har 10 minute pe monday to friday chalega

0 0 1 * * backup.sh   -> Monthly backup job
