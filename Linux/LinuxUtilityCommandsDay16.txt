****************** Basic Unix Tools ************* 	

🔍 find /etc : find all files and folders inside /etc folder

find /etc > etcfiles.txt  : find all files in etc and put the list in etcfiles.txt

find /etc -name passwd    : find passwd files inside etc folder

find / -name myfile.txt   : find myfile.txt in the entire system  (may be slow and gives permission error)

find /home -iname myfile.txt  : case insensitive search

find / -name "file.txt" 2>/dev/null   : find files from / directory and hides errors 

find . -name "file.txt"     : find file.txt in current directory, 👉 ✔ current folder + uske andar sab folders mein search karega

find -name "file.txt"       : same as above command

find /var -name "*.log"       : it will seaarch all .log files 
 
find /home -type f -name "test.txt"    : it will search only files test.txt not directoy ( for directoy use -d)

find / -size +1G      : identifies large files more than 1 GB

find / -size -10k     : identifies files less that 10 kb

find /etc -mtime -2   : recently changes config files

find /var -mtime -1   : last one day modify files

find /var -mtime +7   : 7 days old files

find /etc -maxdepth 1  : depth control means kitni depth tak seaarch karega, 👉 sirf /etc ke andar (andar ke folders ke andar nahi jayega)

find /etc -maxdepth 2   : it will search files till 2 depth means inside one more folder of /etc

find /var -type f -name "*.log" -mtime -1 -size +10M  👉.log files, last 1 day mein modified, size > 10MB

locate myfile.txt     : it is fast because it uses database


************** find without maxdepth vs with maxdepth  *****************************

📂 current structure
. (current dir = level 0)
├── file1.txt              (level 1)  ✅ new file
└── folder1                (level 1)
     └── file1.txt         (level 2)
	 
🔍 Commands:

✅ 1. -maxdepth 0
find . -maxdepth 0 -name "file1.txt"

👉 sirf . check karega (level 0)

❌ current dir ka naam file1.txt nahi hai
👉 no output

✅ 2. -maxdepth 1
find . -maxdepth 1 -name "file1.txt"

👉 check karega:

. (level 0)
uske andar ke entries (level 1)
Output:
./file1.txt

✔ kyunki:

./file1.txt → level 1 ✔
./folder1/file1.txt → level 2 ❌ (ignore)


✅ 3. -maxdepth 2
find . -maxdepth 2 -name "file1.txt"

👉 check karega level 0 → 2

Output:
./file1.txt
./folder1/file1.txt


👉 -maxdepth N =
current directory (level 0) + N levels inside

📊 Easy Table
maxdepth	kya milega
0			sirf .
1			. + direct files/folders
2			+ unke andar ke files
3			+ aur andar


👉
-maxdepth N limits how deep find will search from the starting directory, where current directory is level 0.


👉 -maxdepth 0 ka matlab:

✔ sirf starting path (.) ko check karo
❌ uske andar kuch bhi mat dekho


.                → level 0
./file1.txt      → level 1
./folder1        → level 1
./folder1/file   → level 2

Sirf level 2 chahiye to :
find . -mindepth 2 -maxdepth 2 -name "file1.txt"
✅ Output:
./folder1/file1.txt


level2 se niche sab:
find . -mindepth 2 -name "file1.txt"
