LINUX USER AND LOGIN COMMANDS: When we log in to a Linux VM / Server, we often need to check current user details, logged-in users, and login history.
Below are some important commands used for this purpose.

1. whoami -> current login user

EX: rblashish07@myfirstgcpvm:~$ whoami
rblashish07  (This means the current logged-in user is rblashish07)

2. id -> current user details ( user id, group id, user belong to which group, Useful for permission troubleshooting.)

EX: rblashish07@myfirstgcpvm:~$ id
uid=1002(rblashish07) gid=1003(rblashish07) groups=1003(rblashish07),4(adm),20(dialout),24(cdrom),25(floppy),29(audio),30(dip),44(video),46(plugdev),105(lxd),111(netdev),1000(ubuntu),1001(google-sudoers)

3. who  -> currenlty logged-in Users (who else logged-in to the server in current time, its username, terminal, time, IP address from where login happened)

EX: rblashish07@myfirstgcpvm:~$ who
rblashish07 pts/0        2026-03-14 09:38 (35.235.244.32)

4.  w ->  ( User login + wo kya kaam kar raha hai, Shows logged-in users along with their activity.)

EX: rblashish07@myfirstgcpvm:~$ w
09:53:43 up 41 min,  1 user,  load average: 0.03, 0.02, 0.00
USER     TTY      FROM             LOGIN@   IDLE   JCPU   PCPU  WHAT
rblashis          35.235.244.32    09:38   40:51   0.00s  0.02s sshd: rblashish07 [priv]

5. last ->  Login History (kaun user, kab login hua, kaha se login hua)

EX: rblashish07@myfirstgcpvm:~$ last
rblashis pts/0        35.235.244.32    Sat Mar 14 09:38   still logged in
reboot   system boot  6.17.0-1008-gcp  Sat Mar 14 09:11   still running

wtmp begins Sat Mar 14 09:11:51 2026


Note : When we connect to a VM using SSH, the connection usually goes through an internet gateway or cloud proxy.

Because of this, the IP shown in commands like who or w may be:

✔ GCP SSH Proxy IP
✔ Public NAT IP
